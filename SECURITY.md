# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| `main`  | ✅        |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Email **security@transitx.dev** with:

1. Description of the vulnerability and potential impact
2. Steps to reproduce or proof-of-concept
3. Affected component (frontend / backend / contracts)
4. Suggested mitigations (optional)

You will receive acknowledgement within **48 hours** and a resolution timeline within **7 days**.

## Scope

**In scope:** smart contract fund-release logic, backend JWT auth, wallet key handling, escrow logic.  
**Out of scope:** third-party dependencies (report upstream), issues requiring physical device access.

## Disclosure

We follow coordinated disclosure. A security advisory crediting the reporter will be published after a fix is deployed (anonymity available on request).
